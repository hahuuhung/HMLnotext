# Kế hoạch Triển khai: Trải nghiệm UX No-code Nâng cao

Kế hoạch này tập trung vào việc cải thiện trải nghiệm người dùng (UX) cho giao diện dạng Node nhằm hướng tới một môi trường "No-code" thân thiện theo chuẩn n8n, nhưng vẫn mạnh mẽ cho người dùng chuyên sâu.

## User Review Required

> [!IMPORTANT]
> - Giao diện "Advanced Expression" sẽ được cung cấp dưới dạng một nút Toggle (Công tắc chuyển đổi) trong Inspector. Khi bật, nó sẽ thay thế các input form cơ bản bằng một hộp thoại viết biểu thức/biến (giống n8n). 
> - Nút "Test this node" sẽ giả lập trạng thái "Running" chỉ riêng cho node hiện tại và nếu test thất bại, nó sẽ chuyển sang trạng thái lỗi (error). Bạn đồng ý với cơ chế giả lập "test node" này chứ?

## Phân tích & Thay đổi Đề xuất

### 1. Cấu trúc State (Trạng thái Node)
Sẽ bổ sung thêm một số thuộc tính vào đối tượng `data` của Node:
- `errorMsg?: string` (Mô tả nguyên nhân lỗi & gợi ý sửa)
- `advancedMode?: boolean` (Bật/Tắt chế độ nhập biểu thức chuyên sâu)
- `advancedExpression?: string` (Dữ liệu biểu thức code)

### 2. Thành phần CustomNodes (`src/components/CustomNodes.tsx`)
Hiển thị trạng thái lỗi và cảnh báo trên canvas:
#### [MODIFY] [CustomNodes.tsx](file:///d:/AntiGravity/HML/src/components/CustomNodes.tsx)
- Cập nhật hàm `getStatusColor` và logic giao diện để thêm trạng thái `error`.
- Thêm hộp cảnh báo đỏ (Error Badge / Tooltip) hiển thị icon cảnh báo (AlertTriangle) và `data.errorMsg` khi `data.status === 'error'`.
- Thêm dấu hiệu (Badge) nhỏ màu tím nếu node đang bật `advancedMode`.

### 3. Thành phần Inspector (`src/App.tsx`)
Bổ sung các form, biểu thức nâng cao và kiểm thử Node:
#### [MODIFY] [App.tsx](file:///d:/AntiGravity/HML/src/App.tsx)
- **Presets & Form**: Mở rộng các input trong Inspector dựa trên `subtype`, sử dụng đa dạng các form (dropdown, file upload, color picker, range slider).
- **Advanced Expression Toggle**: Thêm một switch "Chế độ chuyên sâu (Advanced Expression)". Khi bật, các input cơ bản sẽ ẩn đi, nhường chỗ cho một `textarea` nền tối để người dùng viết JS variables (ví dụ: `{{ $trigger.data.url }}`).
- **Test This Node**:
  - Thêm nút **"Kiểm thử Node này (Test Node)"** vào phía dưới cùng của Inspector.
  - Khi bấm, node sẽ chuyển sang `status: 'running'` trong 1-2 giây.
  - Sẽ có xác suất nhỏ ngẫu nhiên (hoặc dựa trên input trống) báo lỗi để demo trạng thái hiển thị Lỗi và gợi ý.
- **Hiển thị Lỗi trong Inspector**: Nếu node có lỗi, Inspector sẽ hiển thị một khối Alert đỏ hướng dẫn cách sửa (Ví dụ: "Thiếu thông số Cron", "Không thể phân tích URL").

## Kế hoạch Xác minh (Verification Plan)

### Thử nghiệm thủ công (Manual Verification)
1. Chọn một Node bất kỳ trên Canvas.
2. Trong bảng Thuộc tính, bật/tắt **"Chế độ chuyên sâu (Advanced Expression)"** để xác minh form chuyển đổi thành code box.
3. Bấm **"Kiểm thử Node này"** -> Chờ xem Node chuyển sang màu cam (đang chạy) và sau đó là xanh lá (thành công) hoặc đỏ (thất bại kèm thông báo nguyên nhân).
4. Xác minh giao diện lỗi hiển thị nổi bật trên Node ở Canvas với nguyên nhân tiếng Việt.
